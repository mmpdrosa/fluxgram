title: Control Flow

Fluxgram includes flow-level control primitives for conversations that should be split into reusable units.

## Branching

```ts
branch(
  (ctx) => ctx.store.plan === "pro",
  send("Pro path."),
  send("Free path."),
  send("Could not evaluate plan."),
);
```

The fourth argument is an optional error arm. If the condition throws and no error arm exists, Fluxgram routes the error through the flow error handler.

## Subflows

Use `callFlow` to run a child flow with an isolated store, then return a value to the caller.

```ts
const collectName = [prompt.text("Name?", { store: "name" }), ret.fromStore("name")];

const profile = fx.flow("profile", [
  callFlow(collectName, { storeResult: "name" }),
  (ctx) => send(`Stored ${ctx.store.name}.`),
]);
```

`ret(value)` exits a subflow early. If a subflow finishes without `ret`, the caller receives `undefined`.

## Continuations

Continuations are named return points inside the current flow. They are useful for menu loops, back buttons, and short detours where the user should return to an earlier choice without starting a new flow.

You usually need only two ideas:

- `storeCC("name")` saves "continue from the next step" under a key.
- `redirectCC("name")` jumps back to that saved point later.

## Menu Loop With storeCC And redirectCC

```ts
const MENU_CC = "menu";

fx.flow("menu", [
  storeCC(MENU_CC),
  prompt.buttons("Choose:", {
    reuseMessage: true,
    buttons: [
      btn("Settings", [send("Settings"), redirectCC(MENU_CC)]),
      btn("Exit", send("Closed.")),
    ],
  }),
]);
```

Read the flow like this:

1. `storeCC(MENU_CC)` saves the point right after itself.
2. The menu prompt is shown.
3. If the user clicks "Settings", Fluxgram sends `"Settings"`.
4. `redirectCC(MENU_CC)` jumps back to the saved point, so the menu prompt is shown again.
5. If the user clicks "Exit", the flow sends `"Closed."` and continues normally.

`reuseMessage: true` is a good fit for this pattern because repeated menu renders edit the existing bot message instead of stacking new menu messages.

## Temporary Return Points With callCC

`callCC(key, body)` saves a continuation, runs `body`, then continues after the `callCC` step unless `body` redirects first.

Use it when a nested block needs an escape hatch back to the caller.

```ts
fx.flow("review", [
  send("Before review."),
  callCC("after-review", [
    send("Inside review."),
    prompt.buttons("Skip the rest?", {
      buttons: [btn("Skip", redirectCC("after-review")), btn("Continue", send("Continuing."))],
    }),
    send("This runs only if the user did not skip."),
  ]),
  send("After review."),
]);
```

In that example, clicking "Skip" jumps to the point after `callCC("after-review", ...)`, so the flow sends `"After review."`. Clicking "Continue" lets the body finish first.

## When To Use Continuations

Use continuations when the flow is still one conversation but needs to jump back to a known point. Menus and "back to main menu" buttons are the clearest case.

Do not use continuations to hide large application state machines. If the conversation has several independent phases, a subflow with `callFlow` and `ret` is usually easier to understand and test.

If `redirectCC` runs before a key has been saved with `storeCC` or `callCC`, Fluxgram treats it as a flow error. Keep continuation keys close to the menu or block that owns them, and prefer constants for shared keys.
