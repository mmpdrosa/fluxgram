---
title: Why Fluxgram
description: Decide when Fluxgram is the right tool for a Telegram bot.
---

Use Fluxgram when your bot has conversation state that should be visible, durable, and recoverable. Use plain grammY when an update can be handled completely in one function.

## Use Fluxgram When

- A conversation spans updates: ask a question, wait, validate, branch, and continue.
- The bot needs durability: prompts, timers, and flow state should survive restarts.
- You need timers or polling with `sleep`, `humanSleep`, or `waitFor`.
- Operators need visibility: list active flows, clear stuck chats, inspect handles, or send debug digests.
- Work starts outside Telegram: cron jobs, webhooks, backend services, or admin tools emit events through `FluxgramClient`.

## Use Plain grammY When

- Commands are stateless and finish inside one update handler.
- You do not need durable prompts, timers, external starts, or recovery sweeps.
- You want to design every storage record and concurrency rule yourself.

## The Core Tradeoff

Fluxgram persists paths into registered flow trees. That is what makes resume durable: a waiting prompt stores a path, and the next update continues from that path.

The cost is that flow shape matters. If you insert, remove, or reorder steps while a conversation is active, old paths can point somewhere else. For production bots, use explicit flow versions and choose a version mismatch policy before deploying shape changes.

## Recommended Adoption Path

1. Start with one multi-step command that currently has hidden session state.
2. Keep `MemoryStorage` while developing and testing the flow shape.
3. Add tests with the harness before moving the flow into production.
4. Switch to durable storage when prompts or timers must survive restarts.
5. Add operations only after the flow is useful: `cancelCommand`, active-flow inspection, debug sinks, and event-driven jobs.

Fluxgram is best when you want conversation logic to read like the product journey while the runtime owns resume, queueing, and recovery.
