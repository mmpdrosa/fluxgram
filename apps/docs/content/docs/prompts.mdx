---
title: Prompts
description: Suspend flows for text replies, whole messages, buttons, and multi-select input.
---

Prompts create durable waiters. When the user replies or clicks a Fluxgram button, the engine resumes the flow from the stored path.

## Text Prompts

```ts
import { ValidationError, prompt } from "fluxgram";

prompt.text("How old are you?", {
  store: "age",
  validate: (_ctx, message) => {
    const age = Number(message.text);
    if (!Number.isInteger(age) || age < 0) {
      throw new ValidationError("Send a whole number greater than zero.");
    }
    return age;
  },
});
```

The validation return value is stored. Throw `ValidationError` to keep waiting and send a retry message.

## Message Prompts

```ts
prompt.message("Send any message.", {
  store: "savedMessage",
});
```

Use message prompts when you need the Telegram message id, text, attachment metadata, or other message fields.

## Button Prompts

```ts
import { btn, prompt, set } from "fluxgram";

prompt.buttons("Pick one.", {
  requireButtonText: "Use one of the buttons.",
  reuseMessage: true,
  buttons: [btn("Alpha", set("choice", "Alpha")), btn("Beta", set("choice", "Beta"))],
});
```

Button prompt children are normal steps. They can set store values, send messages, branch, or call other flows. `reuseMessage` edits the previous bot message in place, which is useful for menu-style flows.

## Multi-Select

```ts
prompt.multiSelect("Choose topics.", {
  store: "topics",
  choices: ["Prompts", "Timers", "Media"],
  submitText: "Done",
  preSelected: ["Prompts"],
});
```

Multi-select state is stored durably, so selected options survive a restart before submit when durable storage is used.

## Timeouts

```ts
prompt.text("Reply within a minute.", {
  store: "answer",
  timeoutSecs: 60,
  onTimeout: send("Timed out."),
});
```

Prompt and multi-select timeout handlers require `timeoutSecs`. Without an `onTimeout` step, a timed-out prompt ends the flow.

## Group Prompts

```ts
prompt.text("Only the person who started this flow can answer.", {
  store: "answer",
  onlyFrom: "initiator",
});
```

Use `onlyFrom` in groups when a prompt should accept replies or button clicks from one user instead of anyone in the chat.
