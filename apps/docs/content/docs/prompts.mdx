---
title: Prompts
description: Suspend flows for text replies, whole messages, buttons, and multi-select input.
---

Prompts create durable waiters. Choose the prompt by what the user should send, not by how you plan to store it.

## Text: User Should Type Words

```ts
import { ValidationError, prompt } from "fluxgram";

prompt.text("How old are you?", {
  store: "age",
  validate: (_ctx, message) => {
    const age = Number(message.text);
    if (!Number.isInteger(age) || age < 0) {
      throw new ValidationError("Send a whole number greater than zero.");
    }
    return age;
  },
});
```

Text prompts accept text messages. Non-text replies keep the flow waiting and ask the user to send text. The validation return value is stored. Throw `ValidationError` to keep waiting and send a retry message.

## Whole Message: User May Send Attachments

```ts
prompt.message("Send a receipt, screenshot, or any message.", {
  store: "receiptMessage",
});
```

Use message prompts when you need attachments, the Telegram message id, text, or other message fields. This is the escape hatch for photos, documents, voice notes, and custom Telegram metadata.

## Buttons: User Should Pick One Action

```ts
import { btn, prompt, set } from "fluxgram";

prompt.buttons("Pick one.", {
  requireButtonText: "Use one of the buttons.",
  reuseMessage: true,
  buttons: [btn("Alpha", set("choice", "Alpha")), btn("Beta", set("choice", "Beta"))],
});
```

Button children are normal steps. They can set store values, send messages, branch, or call other flows. `reuseMessage` edits the prompt message in place, which is useful for menu-style flows.

## Multi-Select: User Should Toggle Choices

```ts
prompt.multiSelect("Choose topics.", {
  store: "topics",
  choices: ["Prompts", "Timers", "Media"],
  submitText: "Done",
  preSelected: ["Prompts"],
});
```

Multi-select state is durable, so selected options survive a restart before submit when durable storage is used.

## Validation

Validation can transform the stored value or reject the answer. Throw `ValidationError` for user-correctable input.

```ts
prompt.text("Email?", {
  store: "email",
  validate: (_ctx, message) => {
    if (!message.text?.includes("@")) throw new ValidationError("Send a valid email.");
    return message.text.toLowerCase();
  },
});
```

## Timeouts

```ts
prompt.text("Reply within a minute.", {
  store: "answer",
  timeoutSecs: 60,
  onTimeout: send("Timed out."),
});
```

Prompt and multi-select timeout handlers require `timeoutSecs`. Without an `onTimeout` step, a timed-out prompt ends the flow.

## Group Scoping

```ts
prompt.text("Only the person who started this flow can answer.", {
  store: "answer",
  onlyFrom: "initiator",
});
```

Use `onlyFrom` in groups when a prompt should accept replies or button clicks from one user instead of anyone in the chat.
