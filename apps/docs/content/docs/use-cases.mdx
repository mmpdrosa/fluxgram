---
title: Use Cases
description: Practical bot workflows that fit Fluxgram's durable flow model.
---

## Multi-Step Onboarding

Collect profile fields, validate each answer, branch based on account type, and resume after a restart without losing the user's place.

## Support Triage

Ask for category, urgency, screenshots, and contact details. Use buttons for structured choices and message prompts for attachments.

## Reminder And Follow-Up Bots

Use `sleep` or `waitFor` to pause work durably and send follow-ups later.

## Group Administration

Run setup flows when the bot is added to a group, becomes admin, loses admin, or the group migrates to a supergroup.

## Internal Operations Bots

Start flows from cron jobs or backend processes with `FluxgramClient`, send cross-process messages, and serialize event handling by key.

## Moderation And Review Tools

Forward source messages, collect reviewer decisions with buttons, pin relevant messages, and send debug digests to an operator chat.
