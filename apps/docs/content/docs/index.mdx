Telegram update handlers are easy until the conversation has memory. A command asks a question, the answer arrives in a later update, a button branches the path, a timer wakes tomorrow, and a deploy happens in the middle. The hidden state becomes the hardest part of the bot.

Fluxgram makes that state explicit. You describe the conversation as a durable flow tree; Fluxgram stores where each chat is in that tree and resumes from the stored path when the next update, timer, or event arrives.

```ts
import { btn, Fluxgram, MemoryStorage, prompt, send, set } from "fluxgram";

const fx = new Fluxgram(bot, { storage: new MemoryStorage() });

fx.command(
  "start",
  fx.flow("onboarding", [
    send("Let's set up your profile."),
    prompt.text("What should I call you?", { store: "name" }),
    prompt.buttons("Pick a role.", {
      buttons: [btn("Builder", set("role", "builder")), btn("Operator", set("role", "operator"))],
    }),
    (ctx) => send(`Ready, ${ctx.store.name}. Your role is ${ctx.store.role}.`),
  ]),
);
```

The flow above is not a callback chain hidden in session state. It is a tree of steps. When the text prompt waits, Fluxgram persists that waiting point. When the button is clicked, Fluxgram runs the button step and continues. If the process restarts between updates and you use durable storage, the same path resumes.

## What Fluxgram Handles

- Durable prompts, buttons, multi-select input, and timers.
- Per-chat queueing so same-chat work runs in order while other chats keep moving.
- Flow shape/version policies for deploys that touch active conversations.
- Storage adapters for Memory, SQLite, Postgres, MongoDB, and Redis.
- Runtime operations: active-flow inspection, cleanup, broadcasts, events, and debug digests.
- Network-free testing through the bundled harness.

## What You Still Own

Fluxgram does not hide Telegram. You still create the grammY bot, choose storage, define command names, write copy, decide permissions, and operate the process. Fluxgram gives those decisions a durable workflow runtime.
